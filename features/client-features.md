# Phase 1 Client Feature 列表与 E2E 测试

> 客户端功能，E2E 通过客户端 UI 操作验证 + 服务端配合断言。
> 客户端：macOS / iOS (SwiftUI, 共享 IMCore Package)

---

## 1. 登录 / 注册

### Feature

- 注册页面：输入 name + password，调用 POST /api/v1/users/register
- 登录页面：输入 user_id + password，调用 POST /api/v1/users/login
- Token 本地持久化，App 启动时自动恢复登录态

### E2E

**E2E-C1.1: 注册流程**

1. 打开 App → 进入注册页面
2. 输入 name="测试用户", password="abc123"
3. 点击注册
4. 断言：注册成功后自动登录，进入会话列表页
5. 断言：本地存储了 token

**E2E-C1.2: 登录流程**

1. 打开 App → 进入登录页面
2. 输入 user_id, password
3. 点击登录
4. 断言：进入会话列表页

**E2E-C1.3: Token 失效自动跳转登录**

1. 手动使 token 过期（修改本地时间或等待过期）
2. 重启 App 或触发任意 API 请求
3. 断言：自动跳转到登录/注册页面

**E2E-C1.4: 错误提示**

1. 登录页输入错误密码
2. 断言：显示错误提示 "用户ID或密码错误"

## 2. 会话列表

### Feature

- 显示用户参与的所有会话（P2P + Group）
- 按最后消息时间倒序排列
- 每条会话显示：名称、头像、最后一条消息预览、未读数、时间
- @mentioned 标记
- 下拉刷新

### E2E

**E2E-C2.1: 会话列表加载**

1. 用户已登录，参与 2 个以上会话
2. 进入会话列表页
3. 断言：显示所有会话，按 last_msg_at 倒序
4. 断言：每行显示 conv name、last_message body、时间

**E2E-C2.2: 未读数显示**

1. 其他用户在 conv_A 发 3 条消息
2. 切换到当前用户的会话列表
3. 断言：conv_A 显示未读红点或数字 "3"

**E2E-C2.3: 最后消息实时更新**

1. 停留在会话列表页
2. 其他用户在 conv_A 发新消息
3. 断言：conv_A 自动移到列表顶部，预览更新为新消息

**E2E-C2.4: 下拉刷新**

1. 会话列表页下拉
2. 断言：刷新后列表数据正确

## 3. 聊天页面

### Feature

- 消息气泡展示（自己发的在右侧，收到的在左侧）
- 文本输入框 + 发送按钮
- 加载历史消息（滚动到顶部自动加载更多）
- 新消息实时流入（通过 WebSocket push）
- 显示发送状态（发送中 / 已发送 / 已读）

### E2E

**E2E-C3.1: 发送文本消息**

1. 打开 P2P 会话聊天页
2. 输入 "hello" → 点击发送
3. 断言：消息气泡出现在聊天区域（右侧）
4. 断言：气泡显示 "已发送" 或 "已读" 状态

**E2E-C3.2: 接收实时消息**

1. 打开与 UserB 的聊天页
2. UserB 发送一条消息
3. 断言：消息气泡自动出现在聊天区域（左侧）
4. 断言：显示 sender_id（P2P 可省略）、时间

**E2E-C3.3: 加载历史消息**

1. 当前会话有 60 条以上消息
2. 滚动到聊天区域顶部
3. 断言：自动加载更多历史消息（触发 GET messages API）
4. 断言：滚动位置保持在加载前的消息附近

**E2E-C3.4: 消息时间分隔线**

1. 消息跨越多天
2. 断言：聊天区域显示日期分隔线

## 4. WebSocket 连接管理

### Feature

- App 启动时自动建立 WS 连接（token 从本地读取）
- 30s Ping 心跳保活
- 断线自动重连（指数退避）
- 重连后自动 SessionRecover + 增量同步

### E2E

**E2E-C4.1: 启动自动连接**

1. 登录后关闭 App
2. 重新打开 App
3. 断言：自动恢复登录态并建立 WS 连接
4. 断言：连接建立后收到在线用户的推送

**E2E-C4.2: 断线重连**

1. 客户端处于聊天页面
2. 断开网络（如切换飞行模式）
3. 断言：UI 提示 "连接已断开" 或类似状态
4. 恢复网络
5. 断言：WS 自动重连成功
6. 断言：重连后自动执行 SyncReq，未读消息补全

**E2E-C4.3: 心跳保活**

1. 客户端连接建立后保持静置
2. 断言：服务端不主动断开（持续 2 分钟以上连接正常）

## 5. 多终端状态

### Feature

- 会话页面显示对方在线状态
- 收到上线/下线通知时 UI 更新

### E2E

**E2E-C5.1: 在线状态显示**

1. 打开与 UserB 的聊天页
2. 断言：顶部显示 "在线" 或 "离线"
3. UserB 登录 → 断言：状态变为 "在线"
4. UserB 断开 → 断言：状态变为 "离线"

**E2E-C5.2: 联系人列表在线状态**

1. 打开联系人列表
2. 断言：每个联系人显示在线/离线状态

## 6. 联系人列表

### Feature

- 显示所有联系人
- 显示在线状态
- 点击联系人可发起聊天

### E2E

**E2E-C6.1: 联系人列表展示**

2. 断言：显示所有联系人，每行显示 name, avatar, 在线状态

**E2E-C6.2: 添加联系人**

3. 断言：添加成功，联系人出现在列表

## 7. 群创建 / 群管理

### Feature

- 创建群页：输入群名 + 选择成员
- 群详情页：查看成员列表
- 群设置：加人 / 踢人 / 退群

### E2E

**E2E-C7.1: 创建群**

1. 点击创建群 → 输入群名 → 选择 2 个以上成员 → 确认
2. 断言：创建成功，进入该群聊天页
3. 断言：聊天页显示系统消息 "xxx 创建了群"

**E2E-C7.2: 群详情 — 成员管理**

1. 进入群详情页
2. 断言：显示群名、成员列表（含 role 标识）
3. 点击添加成员 → 选择用户 → 确认
4. 断言：新成员出现在列表中

**E2E-C7.3: 退群**

1. 在群详情页点击退出群聊
2. 断言：退出后回到会话列表
3. 断言：该群不再出现在会话列表中

## 8. 消息已读状态

### Feature

- 自己发送的消息显示已读状态（单/双勾、已读字样）
- 进入聊天页时自动上报已读（POST read API）

### E2E

**E2E-C8.1: 已读状态显示**

1. UserA 发送一条消息给 UserB
2. UserB 打开该会话的聊天页（触发已读上报）
3. 断言：UserA 侧该消息状态变为 "已读"

**E2E-C8.2: 进入会话自动上报已读**

1. UserA 收到 3 条未读推送
2. UserA 打开该会话聊天页
3. 断言：服务端收到 read API 请求
4. 断言：会话列表未读数清零

## 11. 本地缓存

### Feature

- 会话列表本地缓存（CoreData）
- 消息历史本地缓存
- 离线时可查看已缓存的消息

### E2E

**E2E-C11.1: 离线查看历史**

1. 正常加载会话列表和消息
2. 断开网络
3. 断言：会话列表仍然可见（从缓存读取）
4. 打开已缓存的会话
5. 断言：已加载的历史消息仍然可见
